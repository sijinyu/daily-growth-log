# PRD: 하루한줄 (Daily Growth Log)

> 매일 한 줄 성장 기록 커뮤니티 SaaS

---

## 1. Problem Statement

사람들은 매일 조금씩 성장하고 있지만, 그걸 인식하지 못한다. 거창한 일기나 블로그는 부담스럽고, SNS는 과시적이다. **"오늘 뭐 했지?"라는 질문에 한 줄로 답할 수 있는 공간**이 없다.

### 핵심 가설
- 60자 이하의 짧은 기록이라면 매일 쓸 수 있다
- 포인트/스트릭 같은 가벼운 게이미피케이션이 습관 형성을 돕는다
- 타인과의 경쟁이 아닌, 자기 자신과의 비교가 지속 동기를 만든다
- 레벨별 기능 해금이 성장 동기를 강화한다
- 초대제는 희소성과 바이럴을 동시에 만든다

---

## 2. Solution

**하루한줄** — 매일 한 줄(60자)을 기록하고, 성장 포인트를 쌓고, 조용히 레벨업하는 커뮤니티 앱.

### 핵심 키워드
- **한 줄** — 부담 없는 기록
- **조용한 성장** — 경쟁 없는 동기부여
- **감성 UX** — 차분하고 따뜻한 톤
- **해금** — 성장할수록 열리는 기능
- **초대** — 아무나 못 쓰는 감성

### 기술 스택
| 영역 | 선택 |
|------|------|
| Frontend | React 18 + Vite + TypeScript |
| Styling | Tailwind CSS v4 |
| State | Jotai (client) + TanStack Query (server) |
| Backend | Supabase (PostgreSQL + Auth + Realtime) |
| Auth | 카카오 + 구글 OAuth |
| Deploy | PWA (→ 추후 React Native 전환) |
| Push | Web Push API |

## 3. User Stories

### 온보딩
- **US-01**: 신규 유저로서, 카카오/구글 로그인으로 3초 안에 가입하고 싶다
- **US-02**: 첫 방문 유저로서, 앱이 뭐하는 건지 감성 온보딩으로 30초 안에 이해하고 싶다
- **US-03**: 가입한 유저로서, 닉네임을 설정하고 바로 첫 기록을 남기고 싶다

### 기록 (Core)
- **US-04**: 유저로서, 오늘 한 일을 60자 이하로 빠르게 기록하고 싶다
- **US-05**: 유저로서, 기록을 30초 안에 완료하고 싶다 (빠른 입력)
- **US-06**: 유저로서, 내 기록을 삭제할 수 있지만 수정은 불가하게 하고 싶다 (그 순간의 진정성)
- **US-07**: 유저로서, 자정 전까지만 오늘의 기록을 남길 수 있다 (시간 제한)
- **US-08**: 유저로서, 내 기록을 비공개로 설정할 수 있다
- **US-08-1**: Lv.3+ 유저로서, 기록에 이미지 1장을 첨부할 수 있다 (선택, 5MB/1080px 제한)

### 성장 시스템
- **US-09**: 유저로서, 기록할 때마다 포인트를 받아 성장하고 싶다
- **US-10**: 유저로서, 연속 기록(스트릭)을 유지하면 보너스를 받고 싶다
- **US-11**: 유저로서, 레벨이 올라갈 때 감성적인 축하 메시지를 보고 싶다
- **US-12**: 유저로서, 다음 레벨까지 얼마나 남았는지 진행바로 보고 싶다

### 피드 & 소셜
- **US-13**: 유저로서, 다른 사람의 기록을 피드에서 볼 수 있다
- **US-14**: Lv.2+ 유저로서, 다른 사람의 기록에 "응원" (🌱)을 보낼 수 있다
- **US-15**: 유저로서, 꾸준히 기록한 사람들을 볼 수 있다 (streak 기반 리더보드)
- **US-16**: 유저로서, 부적절한 글을 신고할 수 있다

### 자기 비교
- **US-17**: 유저로서, "지난주보다 N회 더 기록했어요" 같은 자기 비교 피드백을 받고 싶다
- **US-18**: 유저로서, 주간 성장 리포트를 받고 싶다

### 프로필
- **US-19**: 유저로서, 내 닉네임, 레벨, 포인트, 스트릭을 한눈에 보고 싶다
- **US-20**: 유저로서, 내 최근 기록 목록을 볼 수 있다

### 리텐션
- **US-21**: 유저로서, 저녁에 감성적인 푸시 알림을 받아 기록을 떠올리고 싶다
- **US-22**: 유저로서, 앱을 열 때 로그인 포인트를 자동으로 받고 싶다

### 기능 해금 & 초대
- **US-23**: Lv.4+ 유저로서, 초대 코드 3개를 발급받아 지인을 초대할 수 있다
- **US-24**: 초대받은 유저로서, 초대 코드로 가입하면 보너스 포인트(+30)를 받는다
- **US-25**: Lv.1 유저로서, 아직 해금되지 않은 기능(이미지, 응원)에 잠금 UI를 볼 수 있다
- **US-26**: 유저로서, 기능이 해금될 때 감성 축하 메시지를 받고 싶다

## 4. Implementation Decisions

### 4.1 Data Models

```
Users
├── id: uuid (PK)
├── nickname: string (unique, 2-10자)
├── email: string
├── auth_provider: enum (kakao, google)
├── points: integer (default 0)
├── level: integer (default 1)
├── streak: integer (default 0)
├── longest_streak: integer (default 0)
├── last_active_date: date
├── subscription_tier: enum (free, premium) — 미래 수익화용
├── is_private: boolean (default false)
├── push_enabled: boolean (default true)
├── created_at: timestamptz
└── updated_at: timestamptz

Posts
├── id: uuid (PK)
├── user_id: uuid (FK → Users)
├── content: string (max 60자)
├── image_url: string | null (Supabase Storage, Lv.3+ only)
├── is_private: boolean (default false)
├── cheers_count: integer (default 0) — "응원" 수
├── reported: boolean (default false)
├── created_at: timestamptz
└── date: date (기록 날짜, 하루 1개 제한용)

Cheers (응원)
├── id: uuid (PK)
├── post_id: uuid (FK → Posts)
├── user_id: uuid (FK → Users, 응원한 사람)
├── created_at: timestamptz
└── UNIQUE(post_id, user_id) — 중복 응원 방지

Activities (포인트 이력)
├── id: uuid (PK)
├── user_id: uuid (FK → Users)
├── type: enum (daily_login, post, first_post_bonus, cheer_received, streak_3, streak_7)
├── points: integer
├── created_at: timestamptz
└── date: date

Reports (신고)
├── id: uuid (PK)
├── reporter_id: uuid (FK → Users)
├── post_id: uuid (FK → Posts)
├── reason: string
├── status: enum (pending, reviewed, dismissed)
├── created_at: timestamptz
└── UNIQUE(reporter_id, post_id) — 중복 신고 방지

InviteCodes (초대 코드)
├── id: uuid (PK)
├── code: string (unique, 8자리)
├── creator_id: uuid (FK → Users, 발급한 유저)
├── used_by: uuid | null (FK → Users, 사용한 유저)
├── created_at: timestamptz
└── used_at: timestamptz | null
```

### 4.1.1 레벨별 기능 해금

| 레벨 | 해금 기능 | 잠금 UI 카피 |
|------|----------|-------------|
| Lv.1 씨앗 | 텍스트 기록 | — |
| Lv.2 새싹 | 응원(🌱) 보내기 | "새싹(Lv.2)이 되면 응원을 보낼 수 있어요" |
| Lv.3 줄기 | 이미지 업로드 (1장, 5MB, 1080px) | "줄기(Lv.3)가 되면 사진도 남길 수 있어요" |
| Lv.4 꽃봉오리 | 초대 코드 3개 발급 | "꽃봉오리(Lv.4)가 되면 친구를 초대할 수 있어요" |
| Lv.5 꽃 | 프로필 테마 커스텀 | "꽃(Lv.5)이 되면 나만의 색을 입힐 수 있어요" |
| Lv.6 나무 | 추후 결정 | — |

### 4.1.2 초대제 로드맵

```
Phase 1 (런칭): 오픈 가입
  → 누구나 가입 가능, 초기 유저 풀 확보

Phase 2 (100+ 유저): 하이브리드
  → 오픈 가입 유지 + Lv.4 유저에게 초대 코드 3개 발급
  → 초대 코드로 가입 시 보너스 +30
  → "초대받은 사람은 조금 더 빠르게 시작해요"

Phase 3 (1000+ 유저): 초대제 전환 (선택)
  → 오픈 가입 닫기, 초대 코드만
  → 대기열: "현재 N명이 기다리고 있어요"
```

### 4.2 Point System

| 활동 | 포인트 | 조건 |
|------|--------|------|
| 일일 로그인 | +5 | 하루 1회 |
| 기록 작성 | +15 | 건당 |
| 오늘의 첫 기록 | +25 보너스 | 하루 1회 |
| 응원 받기 | +2 | 건당 |
| 3일 연속 스트릭 | +20 보너스 | 도달 시 1회 |
| 7일 연속 스트릭 | +50 보너스 | 도달 시 1회 |

### 4.3 Level System

| 레벨 | 필요 포인트 | UX 카피 |
|------|------------|---------|
| Lv.1 씨앗 | 0 | "씨앗을 심었어요" |
| Lv.2 새싹 | 50 | "새싹이 돋았어요" |
| Lv.3 줄기 | 120 | "줄기가 자라고 있어요" |
| Lv.4 꽃봉오리 | 250 | "꽃봉오리가 맺혔어요" |
| Lv.5 꽃 | 500 | "꽃이 피었어요" |
| Lv.6 나무 | 900 | "나무가 되었어요" |

→ 성장 메타포와 그린 테마를 연결
→ 각 레벨 도달 시 해당 레벨의 기능 해금 + 축하 메시지

### 4.4 Pages & Navigation

```
/ (Home)        — 피드 + 빠른 기록 입력
/write          — 기록 작성 (전체 화면)
/profile        — 내 프로필 + 성장 현황
/leaderboard    — 꾸준한 기록러 (streak 기반)
/onboarding     — 첫 방문 온보딩 (3단계)
/settings       — 알림, 비공개, 계정 설정
```

바텀 내비게이션 (4탭):
```
🏠 홈 | ✏️ 기록 | 🏆 기록러 | 👤 프로필
```

### 4.5 Authentication Flow

```
앱 진입
├── 미로그인 → 온보딩 → 카카오/구글 로그인 → 닉네임 설정 → 홈
└── 로그인됨 → 스트릭 체크 → 일일 로그인 포인트 → 홈
```

### 4.6 Key UX Interactions

**기록 작성 플로우:**
1. 홈 하단 입력창 탭 or ✏️ 탭
2. 전체 화면 입력 (60자 카운터 표시)
3. Lv.3+ → 이미지 첨부 버튼 노출 (선택, 1장, 5MB 제한)
4. "기록 남기기" 버튼
5. 포인트 획득 애니메이션 (+15, 첫 기록이면 +25 추가)
6. 피드로 돌아감

**스트릭 체크 (앱 오픈 시):**
1. `last_active_date` 확인
2. 어제 → streak + 1
3. 어제 이전 → streak = 0 (깨짐)
4. 오늘 이미 접속 → 변동 없음
5. streak 3, 7 달성 시 보너스 + 축하 메시지

**응원 (🌱):**
1. 피드에서 기록 옆 🌱 탭
2. 즉시 반영 (Optimistic UI)
3. 기록 작성자에게 +2 포인트
4. 한 기록당 1회만 가능
